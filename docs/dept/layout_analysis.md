## 0. Главный разворот модели (msg 6883)

Владелец отверг плотный дашборд «всё сразу». Новая суть:

> «На экране не должно быть лишней информации. Если мне нужны задачи или дорожная карта — я открою это ОТДЕЛЬНО. Как приложение Claude: когда открыта сессия — я не вижу ничего, кроме чата. Хочу то же самое, только доработанное под свои нужды.»

Значит — **чат-центричная модель**:
- **Основной рабочий экран = чистая чат-сессия** (как в приложении Claude / Claude Code). Доминирует диалог с ИИ-дворецким «Альфред». Максимум воздуха, минимум хрома.
- **Дорожная карта, задачи, приоритеты, ошибки, файлы, метрики — НЕ висят на экране.** Они вызываются по запросу (drawer / оверлей / вкладка) и по умолчанию скрыты.
- **«Информативно + футуристично» (msg 6880)** теперь = чистый чат в кинематографичном Уэйн-HUD обрамлении (тонкие направляющие-линии, угловые скобки, лёгкое свечение, живая статус-полоска), а вся плотная информация — мгновенно по запросу, не свалена на холст.

Диагноз прежних макетов (`pro_project`, `console_project_dense`) остаётся в силе, но вывод теперь жёстче: проблема была не только в иерархии 5 коробок — **сама идея показывать 5 коробок на рабочем экране ошибочна**. Рабочий экран — это разговор. Данные — за кулисами, вызываются одним движением.

## 1. Принципы (под чат-центричную кино-модель)

### 1.1. Чат — герой кадра, всё прочее — обрамление
Эталон — **приложение Claude / Claude Code**: открыта сессия → виден только разговор. Ширина диалога ограничена читаемой колонкой (≈720–820px), вокруг — воздух. Мы добавляем поверх этого кино-обрамление Уэйна, но НЕ добавляем контента на холст. Композиция кадра: **герой = лента чата по центру**; борт и HUD — тонкая «рама», ведущая взгляд внутрь.

### 1.2. Прогрессивное раскрытие в максимуме (§Things, Arc)
Всё, кроме чата, прячется до запроса. Кнопки вызова данных — тихие, в HUD/борту. По клику — панель выезжает (drawer) или раскрывается оверлеем, по закрытии — экран снова чистый. Это и есть «открою отдельно, когда надо».

### 1.3. Футуристичность достигается обрамлением, не плотностью (msg 6880)
Кино-HUD «боевого пульта Уэйна» строится приёмами, не загромождающими холст:
- **тонкие направляющие-линии (hairlines)** латунью по краям рабочей зоны — ведут взгляд к центру (как разметка прицела/HUD);
- **угловые скобки `⌐ ¬ ⌙ ⌐`** на активной зоне (печать, «прицел») — только на одном фокус-элементе кадра;
- **лёгкое свечение** активных элементов (`box-shadow` латунью, не неон) — глубина и «живость пульта»;
- **живая статус-полоска / телеметрия** тонкой лентой по краю (бегущие данные пода) — «пульт дышит», но не отвлекает от чата;
- **слои/глубина:** фон (тёмный космос-градиент) → борт/HUD (чуть светлее) → лента чата (светлейший слой) — кинематографичная глубина.
Палитра благородная: чёрный `#0b0a07` + латунь `#c6a567`/`#e3c98c`, без неона.

### 1.4. F/Z-паттерн и фокус
Главный герой кадра — лента чата (центр-низ, куда смотрит печатающий). Верх-лево — контекст («Альфред · НЗСН · Сайт»), верх-право — вызов данных и действия. Это даёт «капитанский взгляд»: где я и что говорю — мгновенно.

### 1.5. Чат постоянный везде, данные — вызываемые
В отличие от прошлой версии: теперь чат **доминирует на всех трёх уровнях** (направление и проект — оба чат-центричны). Различие уровней — только в контексте чата и в наборе вызываемых панелей. Главный экран (орбита) — единственный не-чат экран (это карта-обзор), но и он минималистичен.

## 2. ЭКРАН 3 — ПРОЕКТ (главный рабочий, переделан под чат) ⭐

Назначение: разговор с «Альфредом» по конкретному проекту. Substance (карта/задачи/ошибки/файлы/метрики) доступен мгновенно, но скрыт по умолчанию. Сначала — общий каркас, затем 3 варианта МЕХАНИКИ «чат ⇄ детали».

### 2.0. Общий каркас (одинаков для 3A/3B/3C)
```
┌──────────────────────────────────────────────────────────────────────────────┐

├────┬──────────────────────────────────────────────────────────────────────────┤
│борт│  ⌐                                                                    ¬   │ ← угловые скобки активной зоны
│64px│                                                                          │
│    │              «Альфред» приветствие / лента сообщений                     │
│⦿НЗ │              ┌─────────────────────────────────────────┐                │
│⦿К5 │              │  читаемая колонка чата ≈760px, по центру  │                │
│⦿МО │              │  макс воздуха, пузыри: ИИ слева / моё спр.│                │
│… │                │  живой поток действий — карточками В ЛЕНТЕ │                │
│⚙ │                └─────────────────────────────────────────┘                │
│    │  ⌙                                                                    ⌐   │
│    │              ┌───────────────────────────────────────────┐              │
│    │              │ [+] Сообщение Альфреду…           [🎙] [▷] │  ← строка    │
```

Зоны: **слим-борт 64px** (направления) | **чат во всю рабочую ширину** (лента центрирована ≈760px) | HUD 56px сверху | телеметрия 28px снизу.
- HUD: контекст слева («Альфред · НЗСН · Сайт» — всегда видно, кому говоришь), кнопки вызова данных справа, время.
- **Кино-обрамление:** угловые скобки `⌐¬⌙⌐` обрамляют рабочую зону (фокус кадра = чат); hairline-направляющие латунью по краям; телеметрия-полоска снизу — живые данные пода (бегущая), «пульт дышит».
- По умолчанию виден ТОЛЬКО чат. Карта/задачи/ошибки/файлы — за кнопками HUD. Разница вариантов — КАК они появляются.

### Вариант 3A — «Drawer справа» (рекомендация)
Данные выезжают панелью-ящиком справа, чат сжимается влево, не исчезает.
```
до вызова:                          после клика [Дорожная карта]:
┌────┬────────────────────────┐     ┌────┬──────────────────┬───────────────────┐
│борт│  ⌐              чат   ¬ │     │борт│  чат (ужался)    │ ⌐ ДОРОЖНАЯ КАРТА ¬│
│64  │   лента (760, центр)   │ →   │64  │  лента ≈55%      │ ●─●─◉─○─○─○        │
│    │  ⌙ [ ввод ]          ⌐ │     │    │ [ ввод ]         │ стадия 3/6 · 64%  │
└────┴────────────────────────┘     └────┴──────────────────┴───────────────────┘
                                              drawer ≈420px, скользит, [✕] закрыть
```
Зоны: борт 64px | чат 1fr | **drawer 420px (выезжает справа, transition .25s)**.
**Механика:** клик по [Карта]/[Задачи]/[Ошибки]/[Файлы] в HUD → справа выезжает drawer с этой панелью; чат остаётся виден слева (можно продолжать диалог, глядя на данные). Повторный клик или [✕] — drawer уезжает, чат снова во всю ширину. Можно держать открытым один drawer за раз (вкладки внутри drawer переключают карту/задачи/ошибки).
**Футуристичность:** drawer выезжает с hairline-границей и лёгким свечением кромки (как выдвижная панель пульта); угловые скобки перепрыгивают на drawer, отмечая «новый активный слой». Глубина: drawer — слой поверх.
**Фокус кадра:** по умолчанию — чат; при открытом drawer фокус делится «разговор ↔ данные» (оба видны — удобно «обсуждать карту с Альфредом»).
**Почему лучший:** ближе всего к «открою отдельно, но не теряю разговор»; чат не исчезает (можно тут же спросить Альфреда про то, что в панели); одна понятная механика; кинематографичный выезд.

### Вариант 3B — «Полный оверлей» (чистейший чат)
Данные открываются поверх всего экрана крупным оверлеем; чат временно скрыт.
```
до:                              после клика [Дорожная карта]:
┌────┬──────────────────────┐   ┌──────────────────────────────────────────────┐
│борт│   чат во всю ширину  │ → │  ⌐ ДОРОЖНАЯ КАРТА ПРОЕКТА «САЙТ»          ✕ ¬ │
│64  │   (макс воздух)      │   │  ●────●────◉────○────○────○   крупно, со всем │
│    │   [ ввод ]           │   │  датами, описаниями стадий, во весь экран     │
└────┴──────────────────────┘   │  ⌙  [← вернуться к разговору]              ⌐ │
                                 └──────────────────────────────────────────────┘
```
Зоны: чат 100% → оверлей 100% (поверх, затемняет фон), [✕]/Esc возврат.
**Механика:** клик → панель данных раскрывается во весь рабочий экран поверх чата (фон притухает). Каждая панель (карта/задачи/ошибки/файлы) — отдельный полноэкранный вид. Esc/✕ — мгновенно назад в чат.
**Футуристичность:** оверлей «всплывает» с лёгким zoom+glow (кино-переход), угловые скобки очерчивают весь оверлей — «развернули чертёж на столе Уэйна». Максимальная глубина-слой.
**Фокус кадра:** строго один за раз — либо чистый чат, либо одна развёрнутая панель. Никогда не делится.
**Минус:** нельзя смотреть на данные И вести диалог одновременно — переключение модальное. Зато максимально верен «вижу только чат, остальное открываю отдельно».

### Вариант 3C — «Нижний лист + щелевые вкладки» (peek)
Данные выезжают снизу листом (bottom-sheet), можно «подсмотреть» наполовину.
```
до:                               после клика [Задачи] (лист до 50%):
┌────┬──────────────────────┐    ┌────┬──────────────────────────────────────┐
│борт│   чат                │ →  │борт│  чат (сжался вверх ≈50%)             │
│64  │   [ ввод ]           │    │64  │ ─────────── потяни ↕ ───────────────│
│    │ ▔▔▔ вкладки-щели ▔▔▔ │    │    │  ЗАДАЧИ  ① форма ② SEO ③ данные      │
└────┴──────────────────────┘    └────┴──────────────────────────────────────┘
   ↑ тонкие «язычки» снизу: Карта│Задачи│Ошибки│Файлы (peek-вкладки)
```
Зоны: чат 1fr | bottom-sheet 0 → 30/50/90% (тянется вверх).
**Механика:** внизу рабочей зоны — тонкие щелевые вкладки-«язычки». Тап поднимает лист на 50% (видно и чат сверху, и данные снизу), можно дотянуть до 90% или закрыть свайпом вниз. Хорошо ложится на телефон (нативный жест bottom-sheet).
**Футуристичность:** язычки светятся латунью как индикаторы пульта; лист поднимается с hairline-кромкой. «Выдвижной отсек снизу».
**Фокус кадра:** чат сверху всегда частично виден; данные «подглядываются» снизу. Хорош для быстрого «глянул задачи, не уходя из диалога».
**Минус:** на широком Mac-экране вертикальное деление менее естественно, чем боковой drawer; при 50% и чат, и данные ужаты.

**Рекомендация по экрану 3: вариант 3A (drawer справа)** для Mac, с механикой 3C (bottom-sheet) как мобильная адаптация той же логики. 3A даёт лучший баланс: чистый чат по умолчанию + возможность смотреть данные не теряя разговор + кинематографичный выезд. 3B оставить как опцию «развернуть на весь экран» (двойной клик по заголовку drawer → оверлей).

**Почему так:** Владелец говорит с ИИ про стратегию пода, а проекты — под рукой для перехода, но не доминируют. Рельс заменяет прежнюю плотную сетку карточек.

### Вариант 2B — «Чистый чат + проекты вызываемой панелью»
Рельса нет, проекты открываются drawer/оверлеем по [Проекты] (как в 3A/3B). Экран = только чат направления. Максимальный минимализм; список проектов вызывается, когда нужен переход. Минус: чтобы прыгнуть в проект, всегда нужен 1 лишний клик (открыть панель).

**Рекомендация по экрану 2: 2A** — рельс проектов оправдан (переход между проектами пода — частое действие), но узкий и сворачиваемый. Для направлений с 1–2 проектами рельс авто-сворачивается → 2B.

## 5. Ключевой принцип (резюме для фронтенда)

**Чат — единственный герой рабочего экрана. Данные — за кулисами, вызов одним движением.**
1. **По умолчанию виден только чат** (центрированная читаемая колонка ≈760px, максимум воздуха), плюс тихое кино-обрамление Уэйна (скобки, hairline-направляющие, телеметрия-лента, лёгкое свечение).
2. **Карта/задачи/ошибки/файлы/метрики НЕ на холсте.** Кнопки вызова — в HUD; по клику выезжает **drawer справа 420px** (рекоменд.) либо полноэкранный оферлей (опция «развернуть»). По закрытии — снова чистый чат.
3. **Футуристичность = обрамление, не плотность.** Боевой пульт Уэйна делают тонкие линии, скобки-прицел, свечение активной зоны, живая телеметрия — а не заполнение экрана данными.
4. **Контекст всегда виден:** «Альфред · Направление · Проект» в HUD — Владелец всегда знает, кому говорит.
5. **Три уровня = один паттерн:** орбита (обзор) → чат направления (+ рельс проектов) → чат проекта (+ drawer данных). Везде чат доминирует, меняется лишь контекст и набор вызываемых панелей.

## 6. Спецификация сетки/отступов/типографики (Tailwind, тема Уэйн)

Палитра/шрифты из `pro_project.html` сохраняются. Меняется структура — в сторону чистого чата.

### 6.1. Внешняя сетка (CSS Grid)
```
/* Экран проекта (3A) и направления (2A) — по умолчанию */
grid-template-columns: 64px  1fr;                 /* слим-борт | чат во всю рабочую ширину */
grid-template-rows:    56px  1fr  28px;            /* HUD | чат | телеметрия */
/* drawer данных (3A): добавляется 3-й столбец → 64px 1fr 420px, transition .25s */
/* оверлей (3B): position:fixed inset-0, z-50, backdrop dim */
/* рельс проектов (2A): 64px 1fr 220px; сворачивается в 0 */
/* главный (1A): 64px? нет борта — орбита: rows 56px 1fr 28px, чат Координатора = drawer */
```
- Слим-борт: **64px** (направления, иконки-инициалы).
- Лента чата: центрирована, `max-w-[760px] mx-auto`, по бокам — воздух (фон-холст).
- HUD: **56px**. Телеметрия-полоска: **28px** (живая, тихая).
- Drawer данных: **420px**; оверлей: 100%; рельс проектов: **220px**.

### 6.2. Внутренняя верстка чата
```
лента:   flex-col gap-3 (12px) между сообщениями, max-w-[760px] mx-auto, py-8
пузырь ИИ:   слева, bg-panel, rounded-2xl, p-4, text-cream text-[14px]
пузырь мой:  справа, bg-[#2a2114]/border-gold/30, rounded-2xl
карточка-действие ИИ (поток): в ленте, точка-пульс + «агент · что делает», text-[12px] mono
строка ввода: фикс снизу ленты, bg-panel, border-gold/30, rounded-xl, h-14, max-w-[760px]
```

### 6.3. Кино-обрамление (футуристичность, msg 6880)
```
угловые скобки:  4 шт. по углам рабочей зоны, 14px, border-gold opacity .5 (как .mod, но крупнее, на ВСЮ зону а не на блок)
hairline-направляющие:  border-gold/15, 1px, по краям рабочей зоны
свечение активной зоны:  shadow-[0_0_24px_#c6a56722] на активном слое (чат / открытый drawer)
телеметрия-лента:  h-7, font-mono text-[10px] text-mut, divide-x divide-line, бегущие данные пода
глубина-слои:  холст #060504 → борт/HUD #0a0907 → лента чата #0b0a07 panel → drawer/оверлей светлейший +свечение
```

### 6.4. Отступы и радиусы
- Шкала: `4 / 8 / 12 / 16 / 20 / 24`. Между сообщениями 12px, padding пузыря 16px, лента `py-8` (воздух).
- Радиусы: пузыри/карточки `rounded-2xl` (16px); строка ввода `rounded-xl`; кнопки HUD `rounded-lg`; сферы/аватары `rounded-full`. Drawer — `rounded-none` + угловые скобки (печать «активный слой»).
