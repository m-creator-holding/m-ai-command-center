## 0. Как читать документ

По уточнению Владельца (msg 6844) интерфейс имеет **три уровня иерархии**, а не два. Первоначальная «плоская» модель «8 проектов на орбите» пересмотрена: орбита — это уровень **направлений (бизнесов/подов)**, внутри каждого направления — список **проектов**, у каждого проекта — своя рабочая сессия с ИИ. Структура: Главный экран → Направление → Проект.

Дизайн строим так, чтобы **любой из трёх визуальных вариантов** (A «Орбита», B «Ситуационная комната», C «Спокойный продукт») можно было надеть как тему, не переделывая структуру экранов. Поэтому ниже сначала карта экранов (она едина для всех вариантов), потом сценарии, потом дизайн-система с тремя темами, и только в конце — сравнение и рекомендация.

## 1. Информационная архитектура и карта экранов

### 1.1. Трёхуровневая иерархия (уточнение от Владельца, msg 6844)

По новой модели навигация строится по трём уровням. Поды — это **направления** (бизнесы Владельца), а не отдельные проекты; внутри каждого направления — список проектов со своими сессиями ИИ. Это важное смысловое разграничение: в прежней плоской модели сфера на орбите = один проект; теперь сфера = целый бизнес со своей внутренней структурой.

```
УРОВЕНЬ 1 — ГЛАВНЫЙ ЭКРАН (Орбита направлений)
  Сферы = бизнесы/поды холдинга:
  ФБТО · НЗСН · M-OS · Причал · Карьер · Координата 56 · ЖК Дарвин · [+TG-сеть · АЗС — уточняются]
  Список расширяемый: добавить новое направление = добавить новую сферу
           ↓ клик по сфере направления
УРОВЕНЬ 2 — ЭКРАН НАПРАВЛЕНИЯ (один под/бизнес)
  Список проектов внутри направления + чат ИИ по направлению в целом
  Пример (НЗСН): Сайт | ЦС НЗСН | HR-отдел | [+ другие]
  Пример (ЖК Дарвин): МКД-1 | МКД-2 | Инфраструктура | Продажи | [+ другие]
           ↓ клик по карточке проекта
УРОВЕНЬ 3 — ЭКРАН ПРОЕКТА (один конкретный проект)
  Рабочий стол проекта + чат ИИ, заточенный под этот проект, своя память
           ↑ «Назад к направлению»  ↑↑ «К обзору холдинга»

Сквозные (доступны с любого уровня):
  · Глобальный поиск (направление / проект / задача / файл / разговор)
  · Центр уведомлений (что произошло, что ждёт решения — с указанием, в каком направлении/проекте)
  · Индикатор текущего контекста ИИ (постоянно: «Помощник · НЗСН · Сайт»)
  · Быстрый переключатель направления/проекта без подъёма на орбиту
  · Профиль и выход
```

Принцип: **Владелец всегда знает, в каком контексте сейчас работает ИИ.** Два чата существуют независимо: чат направления (стратегический, весь НЗСН) и чат проекта (тактический, только «Сайт НЗСН»). Индикатор контекста — постоянный, крупный, чтобы команда не ушла «не туда».

### 1.3. Полная карта экранов (три уровня)

Всего **41 экран/состояние** в обновлённой структуре. Звёздочкой (★) — MVP-минимум для одного направления + одного проекта.

---

**УРОВЕНЬ 1 — Главный экран (Орбита направлений), 5 экранов**

1. ★ **Орбита — обзор направлений.** Сферы = поды холдинга (ФБТО, НЗСН, M-OS, Причал, Карьер, Координата 56, ЖК Дарвин + будущие). Центральное ядро «ХОЛДИНГ». Размер/цвет/пульсация сферы = здоровье всего направления (агрегирует статус всех проектов внутри). Счётчики вверху: направлений / активных проектов / агентов в работе / задач ждут решения. Бегущая строка внизу — «что прямо сейчас» по всему холдингу. Клик по сфере → уровень 2.

2. **Орбита — наведение/предпросмотр сферы направления.** При наведении — мини-карточка: название направления, сколько проектов внутри, сколько активных агентов, последнее событие. Не уходим с орбиты.

3. **Орбита — фильтр тревог.** Один тап: показать только направления, где есть блокер или ждут решения; остальные сферы притухают. Помогает в «утреннем обзоре» — сразу видно, куда нырять.

4. **Сводка по холдингу (текстовая).** Альтернативный вид орбиты: список направлений с одной строкой статуса каждого и счётчиком проектов. Одинаковые данные, плоско. Для телефона — вид по умолчанию; на Mac — переключаемый.

5. **Чат с Координатором холдинга.** Особый разговор «над всеми направлениями»: «что важного за ночь?», «сравни НЗСН и Карьер», «какие направления ждут моего решения?». Не привязан ни к одному поду. Открывается с орбиты.

---

**УРОВЕНЬ 2 — Экран направления (один под/бизнес), 8 экранов**

*Это новый слой, которого в первоначальном брифинге не было.*

6. ★ **Направление — обзор.** Шапка: иконка + название пода (НЗСН · Металлообработка) + агрегированный статус («5 агентов в работе · 2 блокера»). Два блока: слева — **список проектов внутри** (карточки: имя проекта, статус, последнее событие, кнопка «Открыть»); справа — **чат с ИИ направления** (подключён к контексту всего пода, видит все проекты). Клик по карточке проекта → уровень 3.

7. **Список проектов направления (расширенный).** Когда проектов много (ЖК Дарвин — 5–7 и более): полный список с фильтрами (статус / тип / активность), поиском по имени.

8. **Добавить проект в направление.** Форма: имя, тип, ссылка на папку/репозиторий, начальные настройки. Расширяемость списка проектов внутри пода — через этот экран.

9. **Направление — открытые вопросы (агрегированные).** Все развилки, где ИИ ждёт решения Владельца, собранные со всех проектов данного направления. Владелец обрабатывает их «пачкой» на уровне направления, не прыгая по проектам.

10. **Направление — история разговоров с ИИ.** Прошлые чаты с ИИ на уровне всего пода (не отдельного проекта). Своя память направления.

11. **Направление — активность (агрегированная лента).** Что происходило во всех проектах пода: хронология событий, деплои, правки, действия агентов по всем вложенным проектам.

12. **Профиль направления.** Паспорт пода: что за бизнес, список проектов, команда агентов уровня направления (Директора, Heads), настройки сессии направления.

13. **Добавить новое направление.** Форма для расширения реестра подов: имя, иконка, цвет-акцент (будущий цвет сферы на орбите), список начальных проектов. Расширяемость орбиты — через этот экран.

---

**УРОВЕНЬ 3 — Экран проекта (один конкретный проект), 9 экранов**

14. ★ **Проект — обзор.** Шапка: иконка направления → «/» → имя проекта + статус «N агентов в работе». Хлебные крошки: «НЗСН / Сайт». Метрики (4 плитки). Две колонки: «Задачи» + «Последняя активность». Чат с ИИ, подключённый к этому проекту (правая колонка на Mac, отдельная вкладка на телефоне).

15. **Проект — Задачи.** Полный список задач со статусами, фильтрами, исполнителем-агентом.

16. **Проект — Активность.** Хронология событий только этого проекта.

17. **Проект — Команда агентов (дерево TraceTree).** Живое дерево агентов проекта, статусы, клик в карточку.

18. **Карточка агента.** Роль, зона ответственности, что делает сейчас, история действий, кнопка «дать задачу».

19. **Проект — Файлы и изменения.** Недавно изменённые файлы, кем (агент/Владелец).

20. **Проект — Открытые вопросы к Владельцу.** Развилки по конкретному проекту (карточки «вариант 1 / вариант 2 / отложить»).

21. **Проект — История разговоров.** Прошлые чаты с ИИ по этому проекту. Своя память, изолированная от других проектов и от чата направления.

22. **Профиль проекта.** Паспорт: что за задача, ссылки (сайт, GitHub, папка), команда агентов, настройки сессии проекта.

---

**СЛОЙ ЧАТ С ИИ — оба контекста (направление и проект), 7 экранов/состояний**

*Чат существует в двух контекстах (направление vs проект), но технически — один компонент с разным подключением.*

23. ★ **Чат — основной разговор.** Лента сообщений + строка ввода. Шапка чата: «Помощник · [Направление] · [Проект]» — всегда видно, к какому контексту подключён. Переключатель контекста прямо в шапке.

24. ★ **Чат — живой поток действий.** Внутри ленты или боковой панели: «читаю файл X», «запускаю агента», «делаю правку», «жду подтверждения». На Mac — правая подколонка от переписки; на телефоне — встроено в ленту карточками.

25. ★ **Карточка подтверждения опасного действия.** В ленте: чёткий вопрос + «что произойдёт» на человеческом языке + указание проекта + «Подтвердить» / «Отклонить». Не теряется при обрыве связи.

26. **Чат — переключатель контекста.** Выпадающий список из шапки: выбрать направление (чат пода) или конкретный проект внутри него. Структура списка двухуровневая: «НЗСН (направление) / Сайт / ЦС НЗСН / HR-отдел». Чтобы команда не ушла не туда — смена контекста требует явного выбора, случайно не переключится.

27. **Чат — голосовой ввод (Кардинал).** Кнопка записи → стоп → отправить (manual record-send). Визуально 4 состояния.

28. **Чат — вложение/результат.** Когда ИИ возвращает файл, отчёт, превью — карточка с предпросмотром.

29. **Чат — состояние «работаю, пока Вас не было».** При возврате: сводка по каждому активному контексту (направлению/проекту) — «НЗСН/Сайт: сделано 2, жду подтверждения 1».

---

**СКВОЗНЫЕ И СЕРВИСНЫЕ, 8 экранов**

30. **Глобальный поиск.** Поверх всего: направление / проект / задача / файл / разговор / агент. Результаты сгруппированы: сначала «Направления», потом «Проекты», потом «Задачи/Активность».

31. ★ **Центр уведомлений.** Что произошло и что ждёт решения — с явным указанием «НЗСН / Сайт» или «НЗСН (направление)» для каждого события. Разделение «информация» / «требует решения».

32. **Настройки и доступ — главная.** Список разделов.

33. **Настройки — «красная зона» подтверждений.** Конфигуратор простым языком: деплой / удаление / внешние интеграции — всегда спрашивать.

34. **Настройки — устройства и вход.** Доверенные устройства, биометрия, выход везде.

35. **Настройки — доступ сотрудников (задел RBAC).** Пусто в v1. Каркас: пользователь → роль → права на направление/проект.

36. **Журнал аудита.** Что/когда/в каком направлении/проекте. Фильтр по уровню иерархии.

37. ★ **Вход / биометрия.** Face ID / Touch ID + токен. «Новое устройство — подтвердите с доверенного».

38. **Профиль Владельца.** Устройства, тема оформления, выход.

---

**СИСТЕМНЫЕ СОСТОЯНИЯ, 3 экрана**

39. ★ **Загрузка / переподключение.** «Восстанавливаю связь, догружаю пропущенное».

40. **Пусто / ошибка.** Заглушки для пустого направления/проекта; ошибка действия ИИ.

41. **Оффлайн.** Последнее состояние, пометка «возможно устарело».

---

Итого: **41 экран/состояние**, из них **11 — MVP-минимум** (★) для одного направления + одного проекта: орбита-обзор, экран направления, экран проекта, чат (основной + поток действий + подтверждение), центр уведомлений, вход, загрузка/реконнект.

**Что добавилось по сравнению с первоначальной версией (было 34):**
- +7 экранов уровня «Направление» (экраны 6–13 в новой нумерации) — это принципиально новый слой.
- Чат разделён на два контекста (направление vs проект) — добавлен переключатель контекста (экран 26) и двухуровневый список в нём.
- Центр уведомлений усложнён: теперь каждое событие несёт двухуровневую метку «направление / проект».
- Два экрана расширяемости: «Добавить проект» (8) и «Добавить направление» (13).
- Системные состояния объединены в один экран-группу (было 4, стало 3 — за счёт слияния «пусто» и «ошибка»).

## 2. Пользовательские сценарии Владельца

**Персона.** Мартин, владелец холдинга, ведёт несколько направлений параллельно (каждое — с несколькими проектами внутри), не программист, мыслит бизнес-задачами. Хочет видеть «что происходит» и «что от меня нужно», давать команды простым языком, не запутаться в двух уровнях иерархии. Работает с Mac утром/вечером, с телефона — в течение дня урывками.

### Сценарий 1 — «Утренний обзор» (2–3 минуты, с телефона или Mac)
1. Открывает «Мостик» → биометрия → **Орбита направлений** (экран 1). Сферы = бизнесы.
2. Глазами считывает: пульсируют ли НЗСН, ЖК Дарвин, M-OS (агенты работают); красные — где блокер или ждёт решения. Счётчики вверху — сколько агентов работало ночью по всему холдингу.
3. Если что-то горит — **фильтр тревог** (экран 3): притухают все, кроме проблемных направлений.
4. Альтернатива «дайте читать» — **Сводка по холдингу** (экран 4): «НЗСН — 2 блокера в 1 проекте; ЖК Дарвин — норма; M-OS — 5 агентов».
5. Задаёт вопрос **Координатору холдинга** (экран 5): «что важного за ночь?» — получает сводку по всем направлениям без погружения.

**Что критично:** статус **направления** считывается за 3–5 секунд по цвету/пульсации, без чтения. Орбита — это уровень бизнесов, не деталей. Это главный аргумент за гибрид A+C.

### Сценарий 2 — «Погрузиться в направление, потом в проект» (5–10 минут)
1. На орбите кликает по сфере НЗСН → **Экран направления НЗСН** (экран 6).
2. Видит: список проектов («Сайт», «ЦС НЗСН», «HR-отдел») с их статусами + агрегированная лента событий. Чат ИИ справа уже подключён к НЗСН-в-целом.
3. Спрашивает чат направления: «Что сейчас важнее всего в НЗСН?» — ИИ видит все проекты пода и отвечает сводно.
4. Видит блокер в карточке «Сайт» → кликает → **Экран проекта «Сайт НЗСН»** (экран 14). Хлебные крошки: «НЗСН / Сайт».
5. На экране проекта: метрики, задачи, активность. Чат теперь подключён именно к Сайту, со своей памятью.
6. Видит **открытые вопросы** (экран 20) — развилки именно по Сайту.

**Что критично:** переход орбита→направление→проект плавный, с явными хлебными крошками; смена контекста чата (с направления на проект) — явная, не молчаливая; Владелец в каждый момент знает, с чем говорит ИИ.

### Сценарий 3 — «Дать задачу ИИ и следить за выполнением» (главный сценарий)
1. На экране проекта «Сайт НЗСН» открывает чат (экран 23). Шапка: «Помощник · НЗСН · Сайт».
2. Пишет: «Форма заявки теряет лиды, почини и покажи отчёт по тендерам».
3. ИИ запускает агентов. **Поток действий** (экран 24): «nzsn-frontend правит форму», «nzsn-tender-analyst готовит отчёт» — живьём.
4. ИИ упирается в деплой → **карточка подтверждения** (экран 25) прямо в ленте: «Опубликовать исправленную форму на сайт НЗСН?» — «Подтвердить».
5. Уходит. ИИ продолжает. Пуш: «НЗСН / Сайт: форма исправлена, жду 1 подтверждение».
6. На телефоне → «пока Вас не было» (экран 29): разбивка по контекстам «НЗСН / Сайт: 2 готово, 1 ждёт».

**Что критично:** поток действий — не лог, а «бригада у меня на глазах». Карточка подтверждения всегда называет направление и проект — чтобы Владелец понимал, о чём речь, без погружения в детали.

### Сценарий 4 — «Переключиться между контекстами» (10–15 секунд)
1. Работает в чате проекта «Сайт НЗСН». Вспомнил про ЖК Дарвин.
2. **Быстрый путь**: в шапке чата — **переключатель контекста** (экран 26). Список двухуровневый: «НЗСН (направление) / Сайт / ЦС НЗСН / HR-отдел» и «ЖК Дарвин (направление) / МКД-1 / МКД-2 / ...». Выбирает «ЖК Дарвин (направление)» — чат переключается на контекст всего пода ЖК Дарвин.
3. **Обзорный путь**: «Назад к направлению» → «Назад к орбите» → клик по ЖК Дарвин.
4. Шапка чата сразу меняется: «Помощник · ЖК Дарвин». Разговор с НЗСН не потерян — хранится на сервере, можно вернуться.

**Что критично:** двухуровневый переключатель (направление + проекты внутри) — ключевой новый компонент, которого не было в плоской модели. Он должен быть быстрым и не путать: переключение без случайного срабатывания, явное подтверждение смены контекста ИИ.

### Сценарий 5 — «Подтвердить опасное действие» (10 секунд, с весом)
1. ИИ запрашивает разрешение: деплой, удаление, push --force, внешняя интеграция.
2. **Карточка подтверждения** (экран 25) — в ленте чата, в центре уведомлений (31), пуш на телефон.
3. Карточка: «[НЗСН / Сайт] Опубликовать исправленную форму на nzsn-industrial.ru?» — всегда с указанием контекста, простым языком, с раскрытием «что произойдёт».
4. «Отклонить» — визуально нейтральнее, не первым тапом. «Подтвердить» — акцентный, требует секунды внимания.
5. Обрыв связи — запрос не теряется (§5.4 концепта).

## 3. Дизайн-система

Цель — **одна структура, три темы**. Компоненты и сетка одинаковы; меняется «кожа» (цвета, фон, плотность, акценты). Тема — это набор токенов, переключаемый в профиле (экран 30).

### 3.1. Токены (переменные оформления)

Берём три набора прямо из готовых макетов, чтобы не выдумывать с нуля.

**Цвет — общие смыслы (одинаковы во всех темах, меняются оттенки):**
| Смысл | Назначение | A «Орбита» | B «Ситком» | C «Спокойный» |
|---|---|---|---|---|
| Фон-основа | холст | `#070d20` (космос+звёзды) | `#0a0e17` (сетка) | `#0f1117` (ровный) |
| Фон-панель | карточки, чат | `#0e1a3a` | `#101a2e` | `#16181f` |
| Акцент-бренд | логотип, активное | бирюза→фиолет градиент | циан градиент | `#7c5cff` фиолет |
| Свой/моё (реплика) | сообщения Владельца | `#2a51c4` | `#1f49b8` | `#7c5cff` |
| Успех/онлайн | «готов», деплой ок | `#3ef0a0` | `#36e08a` | `#38d39f` |
| Внимание/работа | «в работе», агент занят | `#ffc24b` | `#ffc24b` | `#ffb02e` |
| Тревога/блокер | блокеры, опасное | `#ff6b81` | `#ff5b7a` | `#ff6b81` |
| Текст-основной | `#e8eefc` | `#dfe6f2` | `#e7e9ee` |
| Текст-вторичный | `#8aa0cc` | `#8a9bc0` | `#8b91a3` |

Единый словарь статусов проекта/агента (3 цвета): **зелёный = в порядке/готов**, **жёлтый = идёт работа**, **красный = блокер/ждёт решения**. Это сквозной язык всех трёх слоёв и всех тем.

**Типографика:**
- Основной шрифт интерфейса: **Inter** (во всех трёх макетах уже он).
- Моноширинный для цифр/технических меток: **JetBrains Mono** (в A и B; в C — отказ от моно ради «спокойствия»). Тема C сознательно прячет моно — это часть её характера.
- Шкала кеглей: 23 (H1 проекта) / 15–16 (заголовки панелей, имя в чате) / 13–14 (тело, сообщения) / 11–12 (метки, статусы) / 10.5 (микро-метки секций).

**Отступы и радиусы:**
- Сетка отступов: 4 / 8 / 14 / 16 / 18 / 22 / 26 px (из макетов).
- Радиусы: 6–9 (мелкие — иконки, метки), 12–15 (карточки, сообщения), 20–30 (пилюли-статусы), 50% (сферы, аватары).
- Колоночная сетка Mac: **сайдбар ~230–248px | центр 1fr | чат ~380–440px** (общая для B и C; A заменяет центр на орбиту).

### 3.2. Библиотека компонентов (единая для всех слоёв и тем)

1. **Статус-точка** (зелёный/жёлтый/красный) — везде, где есть здоровье.
2. **Плитка метрики** — крупная цифра + подпись («3 / Открытых задач»).
3. **Строка задачи** — чекбокс + название + статус-пилюля.
4. **Строка активности** — точка + текст события + время.
5. **Сфера проекта** (слой 1) — круг с инициалами, цвет = бренд проекта, кольцо-пульсация = «идёт работа».
6. **Карточка проекта** (слой 1 список / слой 2 шапка) — иконка + имя + отрасль + статус + 3 цифры.
7. **Пузырь сообщения** — «моё» (справа, акцент) / «ИИ» (слева, панель).
8. **Карточка действия ИИ** (поток) — точка-пульс + «агент · что делает» + время.
9. **Карточка подтверждения** — заголовок-вопрос + человеческое описание + детали(разворот) + 2 кнопки.
10. **Переключатель проекта** — выпадающий список со статус-точками.
11. **Пилюля-статус** (блокер/в работе/готово) — единые цвета.
12. **Шапка слоя** — иконка + заголовок + статус справа.
13. **Строка навигации сайдбара** — иконка-инициалы + имя + счётчик агентов + статус-точка.
14. **Кнопка голоса (Кардинал)** — 4 состояния (покой/запись/обработка/готово).
15. **Заглушки** — пусто / ошибка / оффлайн / загрузка.

### 3.3. Как одна структура держит три темы

- Все цвета — через переменные (CSS custom properties уже есть во всех макетах: `--c`, `--g`). Тема = один файл значений.
- Компоненты не «знают» цвета напрямую — только через смысловые токены (`статус-успех`, `акцент-бренд`).
- **Гибрид, который концепт уже допускает:** Орбита (A) на слое 1 + «Спокойный» (C) на слоях 2–3 — это просто две темы на разных слоях, структура не ломается. Поэтому гибрид не требует отдельной разработки — он выпадает из системы тем бесплатно.
- Различие плотности (B плотнее, C «воздушнее») держим вторым токеном — «режим плотности» (компактный/просторный), независимым от цвета. Так Владелец может выбрать, например, цвета A, но плотность C.

## 4. Адаптация под Mac и мобильное

### 4.1. Mac (основное рабочее место)
- Три колонки: сайдбар проектов | центр (орбита или рабочий стол) | чат справа. Это раскладка макетов B и C.
- Слой 1 «Орбита» на Mac — во весь центр, чат сворачивается в узкую полосу или прячется (на обзоре чат с Координатором по запросу).
- Окно в доке, нативное (Tauri). Глобальный поиск — по горячей клавише.
- Поток действий ИИ — отдельной колонкой/панелью рядом с перепиской (места хватает).

### 4.2. Телефон (контроль на ходу)
Главное правило: **на телефоне три слоя становятся тремя вкладками снизу** — «Обзор», «Проект», «Чат». Жест влево/вправо — переключение.
- **Слой 1 на телефоне:** орбита тяжела для маленького экрана и пальца. По умолчанию показываем **Сводку-список** (экран 4); орбита — опционально, по тапу «3D-вид». Это решает проблему «попасть в мелкую сферу пальцем».
- **Слой 2:** обзор проекта вертикальной лентой: метрики плитками 2×2, ниже задачи, ниже активность. Табы (Задачи/Активность/Агенты) — горизонтальной прокруткой.
- **Слой 3:** чат на весь экран, поток действий — **встроен в ленту карточками** (а не отдельной колонкой, её негде разместить). Подтверждение — крупная карточка на всю ширину, кнопки большие, разнесены.
- **Пуши** — ключевая мобильная ценность: «работа закончена», «жду подтверждения». Тап по пушу → сразу нужный экран.
- **Биометрия** — вход по Face ID/Touch ID.
- Чего на телефоне НЕ делаем в MVP: журнал аудита, детальные настройки, дерево агентов целиком (показываем свёрнуто) — это «для большого экрана».

### 4.3. Что меняется при переходе Mac→телефон (сводка)
| Элемент | Mac | Телефон |
|---|---|---|
| Навигация слоёв | виды одного окна | нижние вкладки + жесты |
| Слой 1 | орбита во весь центр | список-сводка (орбита опционально) |
| Чат + поток действий | две колонки рядом | одна лента, действия карточками |
| Сайдбар проектов | постоянный слева | переключатель сверху / в шапке |
| Подтверждение | карточка в ленте | карточка на всю ширину |
| Поиск/аудит/настройки | полноценно | поиск да, аудит/глубокие настройки — позже |

## 5. Сравнение трёх вариантов для ежедневной работы

Ключевой вопрос Владельца: что выбрать как направление. Сравниваю по тому, что важно для **ежедневной** работы владельца 8 проектов.

### Вариант A — «Орбита» (кинематографичный, сферы)
- **Плюсы:** мгновенное эмоциональное считывание здоровья холдинга по цвету/пульсации; уникальность («продукт, которого нет ни у кого»); прямая преемственность с готовым `m-orbital`; отлично для «утреннего обзора» и для демонстрации.
- **Минусы:** плотность информации низкая — на одном экране мало цифр; попасть в мелкую сферу пальцем на телефоне трудно; для рутинной работы внутри проекта орбита бесполезна (там нужен рабочий стол); красиво, но при ежедневном использовании «вау» приедается.
- **Лучше всего для:** слой 1 (обзор), демонстрация, телефон в режиме «глянул — успокоился».

### Вариант B — «Ситуационная комната» (плотная сетка карточек)
- **Плюсы:** максимум информации на экране — все 8 проектов видны сразу с цифрами и мини-графиками; «командный центр» в буквальном смысле; хорошо для того, кто хочет держать всё под контролем одним взглядом и читать цифры.
- **Минусы:** плотно и «технично» — ближе к дашборду аналитика, чем к спокойному инструменту; легко перегрузить; мини-графики (bars) на старте — украшение без данных (риск «красиво, но пусто»); на телефоне сетка ломается в длинную ленту, теряя смысл «всё сразу».
- **Лучше всего для:** слой 1 как альтернатива орбите для тех, кто любит цифры; большой монитор.

### Вариант C — «Спокойный продукт» (чистый SaaS, Linear-стиль)
- **Плюсы:** самый удобный для **ежедневной работы внутри проекта** — рабочий стол + чат, ничего лишнего; не утомляет при долгом использовании; проще всего в разработке и поддержке; лучше всех переносится на телефон; соответствует принципу Владельца «UX > визуальный полиш» и «простой русский без жаргона»; нет риска «пусто, но красиво».
- **Минусы:** не передаёт амбицию «продукт, которого нет ни у кого» — выглядит как качественный, но привычный SaaS; на слое обзора (8 проектов) проигрывает A/B в «вау» и в скорости эмоционального считывания.
- **Лучше всего для:** слои 2–3 (рабочий стол + чат) — то, где Владелец проводит 90% времени.

### 5.1. Моя рекомендация Владельцу: **гибрид A + C** — и он ещё сильнее подходит к новой иерархии

**Орбита (вариант A) как уровень 1 (выбор направления) + «Спокойный продукт» (вариант C) на уровнях 2 и 3 (направление и проект).** Финальное слово — за Вами.

**Обоснование с учётом трёхуровневой иерархии:**

1. **Орбита стала ещё более органичной.** Раньше на орбите было 8 разнокалиберных «проектов» (сайт НЗСН и весь M-OS рядом — странно). Теперь на орбите 7–9 **бизнесов одного масштаба**: ФБТО, НЗСН, ЖК Дарвин, Причал. Сферы стали смысловыми равными. Орбита как метафора «смотрю на флот направлений» — теперь честная, не вынужденная.

2. **«Спокойный продукт» — для двух рабочих уровней.** Вариант C ложится одинаково хорошо и на экран направления (список проектов + чат с ИИ пода), и на экран проекта (рабочий стол + чат проекта). Это одна «кожа» на оба рабочих уровня, что упрощает разработку и не создаёт визуальных скачков при погружении.

3. **Двухуровневый «Спокойный» не путает.** Риск новой иерархии — Владелец может запутаться «это чат направления или проекта?». Вариант C с его чистой типографикой и явными шапками («Помощник · НЗСН» vs «Помощник · НЗСН · Сайт») решает это лучше, чем плотный B или кинематографичный A, где акцент на «вау», а не на «что сейчас активно».

4. **Вариант B — как альтернативный вид уровня 1.** «Ситуационная комната» с карточками и мини-графиками полезна когда Владелец хочет цифры, а не сферы. Предлагаю оставить как переключаемый вид главного экрана: кнопка «3D-обзор» / «Список» — те же данные, два представления. Основа — всё равно A.

5. **Что это значит для MVP (Фазы 0–2).** Орбита появляется на Фазе 3 (несколько направлений). Поэтому **старт MVP — это уровни 2 и 3 в теме C** (один под, один проект, чат). Рекомендация не удорожает старт; орбита добавляется, когда наполнение смысловое.
